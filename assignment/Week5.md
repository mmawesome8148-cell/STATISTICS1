# 1️⃣ 개념 정리 

## 10. 데이터 탐색과 시각화

```
✅ 학습 목표 :
* EDA의 목적을 설명할 수 있다.
* 주어진 데이터셋에서 이상치, 누락값, 분포 등을 식별하고 EDA 결과를 바탕으로 데이터셋의 특징을 해석할 수 있다.
* 공분산과 상관계수를 활용하여 두 변수 간의 관계를 해석할 수 있다.
* 적절한 시각화 기법을 선택하여 데이터의 특성을 효과적으로 전달할 수 있다.
```

### 10.5 분포 시각화
데이터를 직관적으로 이해할 수 있는 방법이라는 점에서 매우 중요.
분포시각화는 양적 척도인지, 질적 척도인지에 따라 구분해서 그림. 
1) **양적 척도의 시각화 방법**: `막대그래프`, `선 그래프`, `히스토그램`
   + 히스토그램: 구간별 도수를 막대그래프로 표현한 것
     - 각 구간이 bin 
     - 처음에 20개 정도의 구간으로 세세하게 나눠서 분포를 살펴본 다음 조금씩 개수를 줄여가는 방식으로 구간 수를 정함. 
     - 구간이 너무 많으면 보기 어렵고, 너무 적으면 정보 손실이 큼.
<br> 
2) **질적 척도의 시각화 방법**
   - 변수 구성이 단순한 경우: `파이차트`, `도넛차트` -> 분포 정도를 면적으로 표현
   - 변수 구성이 복잡한 경우: `트리맵 차트`(분포의 체계적, 위계구조 표현 가능), `와플 차트`
   

### 10.6 관계 시각화
- `산점도`:**두 개의 연속형 변수 간 관계를 나타냄.** 
    - 극단치를 제거하고 그리는 것이 좋음. 
    - 극단치로 인해 주요 분포 구간이 압축돼 시각화 효율 떨어지기 때문. 
    - 팁) 데이터 오브젝트가 너무 많아서 점들이 서로 겹쳐 보일 때 각 점에 투명도를 주면 밀도도 함께 표현할 수 있음. 
    - 단점) 두 개의 변수 관계만 표현 가능.
- `버블차트`: **세 가지 요소의 상관관계 표현 가능.**
    - 관측치가 너무 많으면 정보전달 효율 떨어짐. (100개 이하 권장)
    - 해석할 때는 원의 지름X,원의 '면적'을 통해 크기를 판단해야. 
    
### 10.7 공간 시각화
- 데이터가 지리적 위치와 관련되어 있으면 실제 지도 위에 데이터를 표현하는 것이 효과적.
- 데이터를 더 직관적으로 파악할 수 있음. 
- **팁)** 스토리라인(분석방향)을 잡고 시각화 방법을 결정해야 정보를 효과적으로 전달함. 
- **종류**
    - `도트맵`: 데이터 개요, 분포 파악에 유리/ 정확한 값 전달엔 적합X
    - `버블맵`: 버블차트를 지도 위에 옮겨둔 것. 비율 비교에 효과적/ 단, 버블 영역 겹치지 않게 해야. 
    - `코로플레스맵`: =단계구분도, 데이터 값 크기, 색상에 따라 지역에 대한 값 시각화./ 정확한 수치로 비교하는 것은 어려움, 큰 지역이 강조될 수 있음을 유의하고 해석해야. 
    - `커넥션맵`: =링크맵, 지리적 관계를 나타내고, 경로를 표현할 수도 있음. 주로 지리적 관계 패턴을 파악하기 위해 사용. 
        -예) 지역 간 무역 관계, 항공 경로, 통신 정보 흐름
    - `플로우맵`: 커넥션맵+시작점과 도착점 표시
    - `카토그램`: 각 지역의 면적을 데이터 값에 비례해서 변형하는 시각화
 

### 10.8 박스 플롯
- 하나의 그림으로 연속형 데이터의 분포, 편향성, 평균, 중앙값 등의 다양한 수치들을 보기 쉽게 정리해줌. 
- 데이터의 대체적인 분포 형태를 확인하기 위해 사용하는 시각화
- `최솟값 극단치`: Q1-1.5*IQR
- `Q1`: 1사분위, 하위 25%
- `Q2`: 중앙값, 하위 50%
- `Q3`: 3사분위, 하위 75%
- `최댓값 극단치`: Q3+1.5*IQR
- 박스 플롯 해석시 항상 데이터 분포도를 떠올려야 함.
